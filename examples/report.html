Function App Pkg Extension Library OS Pkg CVE

Graph Layers

Function — source-level call site (tree-sitter AST)
App Package — pip, npm, Go module, Java Maven, gem, cargo, Composer
Native Extension — compiled .so / .node with ELF symbol imports
Shared Library — ELF .so resolved via ld.so search order
OS Package — dpkg/apk installed package

Edge Types

calls — function invokes a package API
contains — package ships a native extension
links-to — extension links shared library (ELF NEEDED)
owned-by — library file belongs to OS package
depends-on — declared dependency (metadata/lockfile)

CVE Tiers

reachable — static code path from app entry to CVE
linked — ELF binary uses the library, no app trace
installed — present in image, nothing links to it

Evidence Strength

traced — source-level call chain (strongest)
bytecode-import — .pyc import chain
binary-buildinfo — Go module in compiled binary
binary-symbol — function from module found in binary symbol table
symbol-link — imports specific .dynsym symbols
binary-link — ELF NEEDED linkage only
declared-dependency — metadata declares dependency

Reachability Classes

request — HTTP handler (Flask, Express, etc.)
dynamic — AI agent tool dispatch (MCP)
background — scheduled task (Celery, cron)
startup — OCI entrypoint shell script
binary — compiled binary containment only

Controllability

external-input — attacker can reach via HTTP/API
operator-controlled — operator sets config/schedule
internal-only — hardcoded startup logic

Conditions

set — env var present in image → path active
not set — env var absent → path conditional
optional import — try/except ImportError

Symbols

Native extensions show which .dynsym symbols they import from each shared library. Symbol data survives binary stripping. dlopen usage is flagged as a caveat.
